# Citadel Runtime License FAQ

## Is the Citadel Runtime "Open Source"?

No. While the source code is publicly visible and you are free to modify it, the Business Source License (BSL) is not an OSI-approved open-source license because it restricts certain commercial uses (namely, hosting a competing managed service).

The **SDKs and Specs** are fully Open Source (Apache 2.0).

## Can I use Citadel in production internally?

**Yes.** The BSL allows you to use Citadel to govern your own agents within your company. You can self-host the runtime on your own infrastructure for internal use.

## What can't I do?

You cannot take the `apps/runtime` code and sell it as a "Hosted Citadel Service" or a "Governance-as-a-Service" platform to third parties. That is reserved for the official Citadel hosted product.

## Why did you choose this license?

We want to build a sustainable business while keeping our core technology transparent. The BSL ensures that we can continue to invest in Citadel without being undercut by large cloud providers who might host our software without contributing back.

## Does this license ever expire?

**Yes.** On the Change Date (2030-01-01), the code automatically converts to Apache 2.0. This ensures that the code will eventually become fully open source.
