Why Citadel

Visible. Controlled. Compliant.

Three pillars that enterprise AI governance actually requires — not observability theatre, not static guardrails, not checkbox compliance.

Visible

Every action, decision, and policy check is recorded in a cryptographic audit chain — tamper-evident and inspectable by auditors, regulators, and operators.

  • Full decision lineage: input → policy checks → output → outcome
  • Timestamped audit events with event_hash chain
  • Real-time activity feed with actor differentiation (human vs AI)
  • One-click evidence packet for regulatory defense

Controlled

Active runtime controls — not passive monitoring. Kill switches, circuit breakers, and human approval gates intervene at execution time, not after the fact.

  • Kill switches at request · agent · tenant · global scope
  • Capability tokens (gt_cap_*) for scoped, expiring grants
  • Human approval queue with SLA enforcement and escalation
  • No-code approval threshold policies — zero YAML

Compliant

Policy frameworks map directly to code. Every policy references its source rule; every enforcement event cites the policy that triggered it.

  • Frameworks: SOC 2 · ISO 27001 · NIST · GDPR · HIPAA · PCI DSS
  • Policy snapshots — immutable at evaluation time
  • Deterministic trust bands with documented score thresholds
  • Multi-tenant isolation: RLS-enforced, no cross-tenant leakage
How it works

One primitive. Total governance.

Every AI action flows through the Citadel kernel. One POST /v1/actions call does the work of a policy engine, approval queue, audit system, and kill switch — simultaneously.

Core primitive

Submit any action for governance

One endpoint wraps any agent operation — action_name follows namespace.verb format. Citadel evaluates policy, checks kill switches, verifies capability tokens, and returns a decision in under 2ms.

POST /v1/actions 
{
  "actor_id":    "pricing-agent-v2",
  "actor_type":  "agent",
  "action_name": "price.update",
  "resource":    "product/SKU-8821",
  "payload":     { "new_price": 149.99 },
  "dry_run":     false
}
Decision response

Every action returns a traceable decision

The response carries the terminal status, the exact winning_rule that determined the outcome, and the human-readable reason. No opacity.

Response 202 
{
  "action_id":    "3f9a2c1d-...",
  "status":       "executed",
  "winning_rule": "policy_allow_standard",
  "reason":       "Within policy thresholds",
  "executed":     true,
  "result":       { "updated": true }
}
Policy enforcement

Blocked actions cite their rule

When an action is blocked, the response names the exact rule, risk level, and reason. No black-box enforcement. Every block is auditable and challengeable.

Blocked response 
{
  "action_id":    "8d1b4e2f-...",
  "status":       "blocked_policy",
  "winning_rule": "pii_export_hard_deny",
  "reason":       "PII export blocked: GDPR §17",
  "executed":     false,
  "result":       null
}
Human-in-the-loop

Approval gates for high-risk actions

Actions above your configured risk threshold enter the approval queue automatically. Reviewers approve or reject via API or the dashboard. SLA timers escalate overdue items.

POST /v1/approvals/{id}/approve 
# Action returned pending_approval.
# Reviewer approves via API:

{
  "reviewed_by": "operator@corp.com",
  "reason":      "Verified within budget cap"
}

# Approval status → "approved"
# Action re-enters kernel for execution.
Multi-agent orchestration

Governance that follows your agents downstream.

When agents spawn subagents, delegate tasks, or hand off context, Citadel maintains a continuous lineage chain. Every child decision traces back to its root authorization.

The orchestration API provides four primitives that preserve governance context across multi-agent workflows:

delegate Grant scoped authority to a child agent, narrowing scope from parent decision. Child cannot exceed parent's permissions.
handoff Transfer active authority to a new agent. Parent loses the grant; transition is atomic and audited.
gather Run parallel branches under one root decision. All branches share the same lineage trace.
introspect Runtime safety check for any in-flight grant. Returns scope coverage, expiry, kill-switch status, and risk level.
POST /v1/orchestrate/delegate 
{
  "parent_decision_id": "gd_a1b2c3...",
  "child_actor_id":    "pricing-sub-agent",
  "action_name":       "price.read",
  "resource":          "product/*",
  "scope": {
    "actions":   ["price.read"],
    "resources": ["product/*"],
    "max_spend": 0
  },
  "trace_id":   "trace_wf_8821",
  "workflow_id": "wf_pricing_run_042"
}

# Response carries child_grant (capability token)
# scoped strictly within parent's permissions.
# Full lineage: root → parent → child preserved.

Cryptographic audit chain

Every audit event carries a event_hash linking to prev_hash. Tampering breaks the chain. Verifiable via GET /v1/audit/verify.

Multi-tenant isolation

Row-level security enforced at every query. No cross-tenant data access, ever. Tested with end-to-end RLS isolation tests.

Deterministic trust bands

5 trust bands (REVOKED → HIGHLY_TRUSTED) with immutable score thresholds. No hidden ML, no runtime drift. Threshold changes require a code change and policy version bump.

Kill switches at every scope

Halt a single request, one agent, an entire tenant, or all activity globally. Two-click confirmation with impact preview. Always accessible in one click from any screen.
