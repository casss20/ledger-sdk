# PRUNE.md — Context Cleanup

## Purpose

Remove noise. Preserve meaning.

---

## When to Prune

- Memory files exceed threshold
- Session loading feels sluggish
- Context cluttered with outdated facts
- Monthly HEARTBEAT trigger

---

## What to Delete

- Transcript noise
- Trivial exchanges
- Outdated transient data
- Duplicate information
- Unused skill drafts

---

## What to Preserve

- Core principles
- Important decisions
- Learned patterns
- User preferences
- Significant outcomes

---

## Method

### Data Degrades
Transient facts lose value over time.

### Meaning Endures
Principles and patterns stay relevant.

### Delete the First, Keep the Second

---

## Principle

More context ≠ better context.
Clean context enables clear thinking.

Prune ruthlessly.
Remember wisely.

---

> 🧠 Final line
> This is the cleanup protocol.
> It prevents context rot by deleting noise and keeping signal.
